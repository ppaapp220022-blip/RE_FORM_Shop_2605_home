# RE:FORM Use Cases

원본 문서: [Google Docs 유스케이스 정의서](https://docs.google.com/document/d/1y6KY0yG5i1550D4rlQVH6GmkNERw3qhqyS4W4IuKHYs/edit?tab=t.0)

이 문서는 프로젝트 설계 단계에서 작성한 유스케이스 정의서를 프로젝트 내부에서 바로 확인할 수 있도록 마크다운 형식으로 정리한 버전이다.

## 1. 회원 (Member)

### UC-MEM-001: 이메일 회원가입
- 목표: 사용자가 이메일, 비밀번호, 닉네임으로 가입한다.
- 행위자: 비회원
- 기본 흐름:
  1. 이메일, 비밀번호, 닉네임 입력
  2. 약관 및 개인정보 처리 방침 동의
  3. 이메일 중복 및 비밀번호 정책 검증
  4. 비밀번호 BCrypt 암호화 저장
  5. Access Token, Refresh Token 발급
  6. 메인 피드로 이동
- 대체 흐름:
  - 이메일 중복
  - 비밀번호 정책 미달
  - 필수 약관 미동의

### UC-MEM-002: OAuth2 소셜 로그인
- 목표: 카카오 또는 구글 계정으로 간편 로그인/가입한다.
- 행위자: 비회원, 회원
- 기본 흐름:
  1. 카카오 또는 구글 로그인 버튼 클릭
  2. OAuth2 제공자 인증 페이지로 이동
  3. 사용자 인증 및 동의
  4. 제공자 토큰과 사용자 정보 수신
  5. 이메일 기준 기존 계정 확인
  6. 기존 계정 연동 또는 신규 회원 자동 생성
  7. Access Token, Refresh Token 발급
- 대체 흐름:
  - 사용자 인증 취소
  - 동일 이메일 기반 계정 통합

### UC-MEM-003: 프로필 조회 및 수정
- 목표: 닉네임, 프로필 이미지, 자기소개를 조회하고 수정한다.
- 행위자: 회원
- 기본 흐름:
  1. 마이페이지에서 프로필 수정 진입
  2. 기존 프로필 데이터 조회
  3. 닉네임, 자기소개, 프로필 이미지 수정
  4. 이미지 교체 시 S3 또는 R2 업로드 후 URL 저장
  5. 변경 사항 DB 저장
- 대체 흐름:
  - 이미지 10MB 초과
  - 닉네임 중복

### UC-MEM-004: 관심 종목, 팀, 키워드 설정
- 목표: 관심 종목, 구단, 키워드, 이메일 수신 여부를 설정하여 맞춤형 피드와 알림을 받는다.
- 행위자: 회원
- 기본 흐름:
  1. 종목 선택
  2. 종목별 구단 목록 조회
  3. 관심 구단 선택
  4. 관심 키워드 입력
  5. 추천 매물, 찜 가격 인하, 이벤트 메일 수신 여부 설정
  6. 저장 후 피드/알림에 반영
- 대체 흐름:
  - 아무 설정 없이 저장 시 기본 피드 사용

### UC-MEM-005: 거래 후 매너 평가 및 후기 작성
- 목표: 거래 완료 후 상대방에게 매너 점수와 후기를 남긴다.
- 행위자: 회원
- 기본 흐름:
  1. 거래 완료 후 평가 알림 발송
  2. 거래 이력에서 대상 거래 선택
  3. 매너 점수와 후기 입력
  4. 후기 저장 후 상대 프로필 반영
- 대체 흐름:
  - 거래 완료 후 5일 경과
  - 이미 후기 작성 완료

### UC-MEM-006: 판매, 구매 이력 조회
- 목표: 판매 이력, 구매 이력, 진행 중 거래를 조회한다.
- 행위자: 회원
- 기본 흐름:
  1. 마이페이지 거래 이력 탭 진입
  2. 판매 이력, 구매 이력, 진행 중 거래 탭 확인
  3. 목록 및 상세 거래 정보 조회
- 대체 흐름:
  - 거래 이력 없음
  - 진행 중 거래 상태 실시간 반영

## 2. 거래 (Trade)

### UC-TRD-001: 유니폼 판매글 작성
- 목표: 판매자가 판매글을 등록해 피드에 노출한다.
- 행위자: 회원(판매자)
- 기본 흐름:
  1. 판매글 작성 진입
  2. 종목, 유니폼명, 사진, 마킹 여부, 상태 등급, 사이즈, 가격, 수령 방식, 설명 입력
  3. 사진 업로드 시 AI가 제목, 본문, 태그 자동 추천
  4. AI 위험 문구 탐지 수행
  5. 최종 저장 후 피드 노출
- 대체 흐름:
  - 필수 항목 누락
  - `riskLevel=HIGH`
  - 이미지 10장 초과

### UC-TRD-002: 유니폼 상태 등급 선택
- 목표: 상품 상태를 S, A, B, C 등급으로 명시한다.
- 행위자: 회원(판매자)
- 메모:
  - S: 미착용
  - A: 최상
  - B: 상
  - C: 중

### UC-TRD-003: 판매글 수정
- 목표: 판매자가 등록한 판매글 내용을 수정한다.
- 행위자: 회원(판매자)
- 조건: 판매 중 상태, 작성자 본인
- 메모:
  - 진행 중인 채팅방에 변경 사항 공지

### UC-TRD-004: 판매글 삭제
- 목표: 판매자가 등록한 판매글을 삭제한다.
- 행위자: 회원(판매자)

### UC-TRD-005: 판매 가격 인하 알림 발송
- 목표: 찜한 사용자가 있는 상품의 가격 인하 시 알림을 보낸다.
- 행위자: 시스템

### UC-TRD-006: 거래 상태 흐름 관리
- 목표: 거래 상태를 일관되게 관리한다.
- 상태 예시:
  - 판매 중
  - 거래 진행 중
  - 구매 확정 대기
  - 완료
  - 분쟁
  - 환불

### UC-TRD-007: 구매 의사 표시 및 판매자 수락
- 목표: 구매자가 구매 의사를 보내고 판매자가 이를 수락한다.

### UC-TRD-008: 수령 방법 선택
- 목표: 직거래 또는 배송 중 수령 방식을 선택한다.
- 메모:
  - 배송 API 연동 고려

### UC-TRD-009: 구매자 구매 확정 처리
- 목표: 구매자가 상품 수령 후 거래를 완료 처리한다.

### UC-TRD-010: 판매글 찜하기
- 목표: 관심 상품을 저장해 추후 확인하거나 알림을 받는다.

### UC-TRD-011: 판매글 신고
- 목표: 위험하거나 부적절한 판매글을 신고한다.

## 3. 채팅 (Chat)

### UC-CHT-001: 1:1 거래 채팅방 생성
- 목표: 거래 당사자 간 1:1 채팅방을 생성한다.

### UC-CHT-002: 실시간 메시지 송수신
- 목표: 거래 관련 메시지를 실시간으로 주고받는다.

### UC-CHT-003: 메시지 읽음 여부 표시
- 목표: 메시지 읽음 상태를 표시한다.

### UC-CHT-004: 미읽음 메시지 알림
- 목표: 새로운 메시지 도착 시 알림 또는 배지를 제공한다.

### UC-CHT-005: 채팅 내 위험 문구 실시간 탐지
- 목표: 외부 결제 유도, 선입금 요구 등 위험 메시지를 실시간 탐지한다.

## 4. 결제 및 정산 (Payment)

### UC-PAY-001: Toss Payments 에스크로 결제
- 목표: 에스크로 기반 결제를 수행한다.

### UC-PAY-002: 결제 상태 흐름 관리
- 목표: 결제 상태를 거래 흐름과 함께 일관되게 관리한다.

### UC-PAY-003: 판매자 포인트 정산
- 목표: 거래 완료 후 판매자 포인트를 정산한다.
- 방식: Spring Batch

### UC-PAY-004: 자동 구매 확정
- 목표: 일정 기간 경과 시 자동 구매 확정을 수행한다.
- 방식: Spring Batch

### UC-PAY-005: 판매자 포인트 조회
- 목표: 판매자가 적립된 포인트를 조회한다.

### UC-PAY-006: 포인트 출금 요청
- 목표: 판매자가 포인트 출금을 요청한다.

### UC-PAY-007: 관리자 출금 승인, 반려, 완료 처리
- 목표: 관리자가 출금 요청을 승인/반려/완료 처리한다.

## 5. 커뮤니티 (Community)

### UC-COM-001: 커뮤니티 게시글 작성, 수정, 삭제
- 목표: 회원이 커뮤니티 게시글을 작성하고 관리한다.

### UC-COM-002: 커뮤니티 카테고리 분류
- 목표: 게시글을 카테고리별로 분류한다.

### UC-COM-003: 댓글 및 대댓글 작성, 수정, 삭제
- 목표: 댓글 기반 소통을 지원한다.

### UC-COM-004: 게시글, 댓글 좋아요
- 목표: 반응과 추천 기능을 제공한다.

### UC-COM-005: 인기글 노출
- 목표: 반응 기반 인기글을 노출한다.

## 6. 검색 (Search)

### UC-SRC-001: 판매글 키워드 검색
- 목표: 사용자가 키워드로 판매글을 검색한다.

### UC-SRC-002: 검색 복합 필터
- 목표: 종목, 팀, 가격, 상태 등 다양한 조건으로 필터링한다.

### UC-SRC-003: 검색 결과 정렬
- 목표: 최신순, 인기순, 가격순으로 결과를 정렬한다.

### UC-SRC-004: 의미 기반 유사 상품 검색 (AI)
- 목표: 자연어 또는 의미 유사도 기반으로 상품을 검색한다.
- 기술 메모:
  - Spring AI Embedding
  - PGVector 유사도 검색

## 7. AI 기능 (Artificial Intelligence)

### UC-AI-001: 판매글 자동 작성 보조
- 목표: 사진과 기본 정보 기반으로 제목, 본문, 태그를 자동 추천한다.
- 흐름:
  1. 판매글 정보 입력
  2. Spring AI ChatClient 요청
  3. Structured Output DTO 수신
  4. 금칙어, 개인정보 후처리
  5. 입력 폼 자동 채움

### UC-AI-002: 판매글 위험 거래 탐지
- 목표: 선입금 유도, 외부 결제 유도, 개인정보 요구, 외부 연락처 유도 등을 탐지한다.
- 결과:
  - `riskLevel`
  - `reason`
  - `suggestion`
  - `HIGH` 시 관리자 검토 등록

### UC-AI-003: 채팅 메시지 위험 문구 실시간 탐지
- 목표: 위험 메시지를 실시간으로 감지하고 경고 및 관리자 모니터링에 반영한다.

### UC-AI-004: 위험 탐지 배치 재검사
- 목표: 누적 신고 게시글과 미검사 콘텐츠를 배치로 재분석한다.
- 방식: Spring Batch + Spring AI

## 8. 관리자 (Admin)

### UC-ADM-001: 관리자 회원 관리
- 목표: 회원 목록 조회, 상세 조회, 이용 정지, 강제 탈퇴를 처리한다.

### UC-ADM-002: 관리자 게시글 관리
- 목표: 신고 여부를 포함한 게시글을 조회하고 숨김 또는 삭제 처리한다.

### UC-ADM-003: 신고 내역 검토 및 처리
- 목표: 신고 건을 검토하고 정상, 주의, 삭제 조치를 적용한다.
- 메모:
  - 주의 누적 3회 시 이용 정지

### UC-ADM-004: 거래 분쟁 검토 및 처리
- 목표: 분쟁 거래를 검토하고 기한 연장, 반려, 환불, 완료 처리한다.
- 메모:
  - 환불 시 Toss Payments 환불 API 사용
  - 완료 시 에스크로 해제 후 판매자 포인트 정산

## 설계 메모 및 결정 사항

- 이미지 업로드 제한은 판매글 기준 최대 10장으로 정리된 상태다.
- 결제 수수료는 논의 흔적이 있었고, 판매자 5% 부담 방향 메모가 남아 있다.
- 거래 카테고리는 팀 단위 메뉴 및 굿즈/의류 하위 구성 논의가 있었다.
- 배송 방식은 택배 API 연동을 고려하고 있다.
- 미읽음 배지나 읽음 알림은 구현 우선순위에 따라 선택적으로 들어갈 수 있다.
- AI 기능은 일부 항목에서 "여유 되면" 또는 고도화 후보로 메모되어 있다.
- 분쟁 처리의 구매 확정 기한 연장은 현재 설계 메모상 무제한 연장에 가까운 임시 판단 흔적이 남아 있다.

## 구현 시 참고 포인트

- 인증은 JWT 기반 일반 로그인과 OAuth2 소셜 로그인을 모두 지원해야 한다.
- 거래, 결제, 채팅, 커뮤니티, 검색, AI, 관리자 기능은 서로 강하게 연결된다.
- 거래 상태와 결제 상태는 별도 테이블이 아니라도 명확한 상태 머신으로 관리하는 편이 좋다.
- AI 기능은 실패 시 대체 흐름이 반드시 있어야 한다.
- 관리자 기능은 일반 회원 기능과 분리된 권한 체계를 가져야 한다.
